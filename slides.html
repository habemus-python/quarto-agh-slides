Part 1: Markdown table alignment

Quarto supports normal Markdown tables.

Alignment is controlled with colons in the separator row:

| Label | Value | Unit |
|:------|------:|:----:|
| mass  | 1.23  | kg   |
| time  | 12.5  | s    |

Result:

Label Value Unit
mass 1.23 kg
time 12.5 s

Part 2: Generate tables from Python

A pandas.DataFrame can be displayed directly in Quarto.

This is useful when the table is produced by calculations rather than typed manually.

raw[["sample", "mass_kg", "time_s"]]
sample mass_kg time_s
0 A 1.234560 0.0123
1 B 12.000000 15.6780
2 C 0.004321 1234.5000
3 D 1234.567000 89.0010

Part 3: Number formatting

The section will explain common Python format specifiers:

Specifier Meaning Example
.2f 2 digits after decimal point 12.35
.3g 3 significant digits 0.00432
.2e scientific notation 1.23e-04
,.2f thousands separator 1,234.57
sample mass_kg time_s measured_at uncertainty
0 A 1.23 0.01 09:15 1.23e-04
1 B 12 15.68 09:45 4.57e-02
2 C 0.00432 1234.50 10:30 1.23e+01
3 D 1.23e+03 89.00 11:05 9.88e-06

Part 4: Significant digits

A key point will be the difference between decimal places and significant digits.

f"{x:.3f}"  # 3 decimal places
f"{x:.3g}"  # 3 significant digits

This matters for measurements, where values may have very different magnitudes.

raw value 3 decimal places 3 significant digits
0 1234.567000 1234.567 1.23e+03
1 12.345670 12.346 12.3
2 0.123457 0.123 0.123
3 0.000123 0.000 0.000123

Part 5: Time values

Time data should usually be converted from full timestamps to a format that matches the purpose of the table.

Examples:

Format Meaning
%H:%M hour and minute
%Y-%m-%d date
%Y-%m-%d %H:%M date and time
sample measured_at time only date only date and time
0 A 2026-05-14 09:15:00 09:15 2026-05-14 2026-05-14 09:15
